# 20 因子槽 0.3.5

这是 Reloaded-II 独立版，不依赖也不会修改 Luma、ReShade 或游戏存档。战斗注入逻辑继续使用已选定的 `0.3.0-test7` 基线；0.3.5 在 UTF-16 IME 提交和 DBCS 消息转换之外，增加了搜索缓冲区的可逆 GBK 乱码修复。

## 安装与使用

1. 完全退出游戏。
2. 若已经使用旧版，先备份旧目录中的 `GBFR-ExtraSigilSlots20.ini`。
3. 删除旧的 `GBFR.ExtraSigilSlots20.Reloaded` 文件夹，再安装本包；不要覆盖混装。
4. 如需保留原来的虚拟槽选择，把备份的 INI 放回新目录。本包不携带默认 INI，因此不会覆盖用户选择。
5. 通过 Reloaded-II 启动游戏，在装备界面按 `F8` 打开窗口。旧配置中的默认 `NumPad 8` 会迁移为 `F8`；用户此前设置的其他按键会保留。

## 界面

- `中文` / `English`：切换界面和因子名称语言。
- 因子搜索栏：中文输入法提交时优先通过 `ImmGetCompositionStringW` 取得 UTF-16；若消息仍被旧后端按西欧字符写入缓冲区，则自动把可逆序列修回中文，例如 `ÔÚ → 在`、`ÖÐÎÄ → 中文`。修复后会重载并重新聚焦输入框；直接粘贴的正确中文不会改变。
- 当前角色：显示人物名，不显示角色 hash 或内存地址。
- 刷新因子：重新读取游戏主因子库存。
- 扫描出因子数量：所有非空、未禁用的物理因子记录总数，包括当前已装备而不会出现在选择列表中的记录。
- 当前状态：只显示“当前状态可修改”或“当前状态不可修改”。
- 13–20：八个额外虚拟槽。

界面不再显示调试用的手动 Apply/AutoApply 控件；从 0.3.1 起会将旧配置中的 `AutoApply=0` 迁移为自动应用，确保选择或清空槽位后仍会立即请求安全重建。

选择弹窗会继续隐藏所有已被原生装备槽占用的因子，并过滤掉不属于当前角色的专属因子。因此“匹配的可选因子”数量通常小于“扫描出因子数量”；前者是当前可选/搜索结果，后者才是完整扫描总数。

MOD 不写 SaveData，不修改 `GemData.WORN_BY`。如果一个已选虚拟因子后来被装入原生 1–12 槽，刷新库存时会安全清除对应虚拟选择。
