# GBFR 扩展因子槽 0.5.0

这是 Reloaded-II 独立版，不依赖也不会修改 Luma、ReShade 或游戏存档。战斗注入逻辑继续使用已选定的 `0.3.0-test7` 基线；0.5.0 支持通过 INI 启用 1–24 个扩展槽，并保留多角色命名预设、因子占用筛选和安全转移功能。

## 安装与使用

1. 完全退出游戏。
2. 若已经使用旧版，先备份旧目录中的配置 INI；若已经创建预设，也备份 `GBFR-ExtraSigilSlots20.presets.json`。
3. 删除旧的 `GBFR.ExtraSigilSlots20.Reloaded` 文件夹，再安装本包；不要覆盖混装。
4. 如需保留原来的虚拟槽选择和预设，安装后把备份的 INI 与预设 JSON 放回新目录；旧版的 `GBFR-ExtraSigilSlots20.ini` 必须先改名为 `GBFR-ExtraSigilSlotsNumConfig.ini`。本包携带新名称的默认 INI，方便直接修改扩展槽数量；运行时创建的预设 JSON 仍不会打包，因此不会覆盖命名预设。
5. 通过 Reloaded-II 启动游戏，在装备界面按 `F8` 打开窗口。旧配置中的默认 `NumPad 8` 会迁移为 `F8`；用户此前设置的其他按键会保留。

## 界面

- `中文` / `English`：切换界面和因子名称语言。
- 因子搜索栏：中文输入法提交时优先通过 `ImmGetCompositionStringW` 取得 UTF-16；若消息仍被旧后端按西欧字符写入缓冲区，则自动把可逆序列修回中文，例如 `ÔÚ → 在`、`ÖÐÎÄ → 中文`。修复后会重载并重新聚焦输入框；直接粘贴的正确中文不会改变。
- 输入捕获：窗口打开时只拦截键盘和鼠标；XInput、DirectInput 与 Raw HID 手柄输入继续原样传给游戏，关闭窗口也不会等待手柄回中。
- 当前角色：显示人物名，不显示角色 hash 或内存地址。
- 刷新因子：重新读取游戏主因子库存。
- 扫描出因子数量：所有非空、未禁用的物理因子记录总数，包括当前已装备而不会出现在选择列表中的记录。
- 当前状态：只显示“当前状态可修改”或“当前状态不可修改”。
- 战斗提示：显示“游戏不支持战斗状态热更新因子，请谅解”；需要在装备界面完成修改。
- 多预设：每个自定义名称的预设保存所有已支持角色当前启用的虚拟槽；可以套用、覆盖、另存、重命名和删除。
- 搜索条件：`所有因子 / 已使用的因子 / 本体已使用的因子 / 扩展已使用的因子 / 未被使用的因子`，默认选择“未被使用的因子”。
- 已使用标注：显示实际角色名、本体或扩展占用位置，以及引用当前因子的预设名称。
- 扩展槽：默认启用 8 个（13–20）；实际数量由 INI 的 `VirtualSlotCount` 决定。

`VirtualSlotCount` 默认是 `8`，必须重启游戏后生效。缺失、空值、非数字、带正负号、负数、小数或其他非法内容会自动改写为 `8`；`0` 会改写为 `1`；超过 `24` 会改写为 `24`。缩小数量时，超出新范围的旧选择会被清空并回写，避免隐藏槽继续占用库存因子。

界面不再显示调试用的手动 Apply/AutoApply 控件；从 0.3.1 起会将旧配置中的 `AutoApply=0` 迁移为自动应用，确保选择或清空槽位后仍会立即请求安全重建。

选择弹窗默认隐藏所有已被本体或扩展槽占用的因子，并继续过滤不属于当前角色的专属因子。切换搜索条件后可以查看已使用的因子：本体已使用项只能查看提示，必须先在对应角色的本体装备栏脱除；扩展已使用项会显示“是/否”确认框，并可勾选“当前菜单期间不再提示”。确认转移后，原角色当前扩展槽以及所有相关预设位置会直接清空。

套用预设时会一次性处理所有角色。若预设引用的物理因子已经被任意角色的本体槽占用，只有对应虚拟槽会变为空并显示“当前因子被用在XX角色上，请您替换另一个因子”，其余槽位仍继续套用。

MOD 不写 SaveData，不修改 `GemData.WORN_BY`。如果一个已选虚拟因子后来被装入原生 1–12 槽，刷新库存时会安全清除对应虚拟选择。
